# Trigger

New feature request

# Steps

1. `project-manager` reads current docs and `.openclaw/context`.
2. Ask clarification questions across:
   - product goal
   - users
   - daily todo flow
   - feature list
   - data/schema
   - design
   - technical constraints
   - deployment
3. Create a shared-understanding summary.
4. Wait for approval from the coordinator.
5. Update `docs/product-requirement.md`, `docs/architecture.md`, and `docs/design-system.md`.
6. Generate small task files in `docs/tasks`.
7. Create Trello cards in `Backlog` using `.openclaw/context/trello-automation.md`.
8. Move implementation-ready tasks to `Code Inprogress` when assigned to `fullstack-developer`.
